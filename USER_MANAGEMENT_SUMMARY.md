#### 1.2 用户管理CRUD接口
- **创建用户**：`POST /users`
  - 验证用户名、手机号、邮箱唯一性
  - 密码加密存储
  - 支持设置角色、上级等字段

- **查询用户列表**：`GET /users`
  - 支持关键词搜索（姓名、用户名、电话、邮箱）
  - 支持角色筛选
  - 支持状态筛选
  - 支持分页
  - 支持排序
  - 自动关联上级名称

- **查询用户详情**：`GET /users/:id`
  - 返回用户完整信息
  - 包含上级名称

- **更新用户**：`PUT /users/:id`
  - 支持更新所有字段
  - 验证唯一性
  - 支持密码更新

- **删除用户**：`DELETE /users/:id`
  - 软删除用户

- **重置密码**：`POST /users/:id/reset-password`
  - 管理员重置用户密码

- **更新状态**：`PATCH /users/:id/status`
  - 更新用户状态（正常、禁用、锁定）

- **获取销售人员**：`GET /users/sales`
  - 获取所有销售人员列表

- **获取下属**：`GET /users/subordinates`
  - 获取当前用户的下属列表

- **获取上级**：`GET /users/superior`
  - 获取当前用户的上级信息

## 权限分配说明

### 角色权限映射

每个角色都有预定义的权限集合：

1. **管理员（admin）**：拥有所有权限
2. **总裁（ceo）**：大部分业务权限，但不能管理用户和角色
3. **技术副总（cto）**：项目管理相关权限
4. **营销副总（cmo）**：销售和客户管理权限
5. **销售经理（sales_manager）**：团队管理权限
6. **销售（sales）**：基础销售权限
7. **项目经理（project_manager）**：项目管理权限
8. **商务（business）**：商务相关权限
9. **财务（finance）**：财务和合同权限

### 权限验证流程

1. 用户登录后，JWT token中包含用户角色信息
2. 访问API时，`PermissionsGuard` 检查用户角色
3. 根据角色获取对应的权限列表
4. 验证用户是否拥有所需权限
5. 如果没有权限，返回403 Forbidden

## 使用说明

### 初始化角色数据

运行以下命令初始化系统角色：

```bash
ts-node scripts/init-roles.ts
```

### API权限要求

- 用户管理相关接口需要对应的权限：
  - `user_create`：创建用户
  - `user_view`：查看用户
  - `user_edit`：编辑用户
  - `user_delete`：删除用户
- 角色管理相关接口需要：
  - `role_view`：查看角色
  - `role_edit`：编辑角色

### 前端使用

1. 用户管理页面：`/system/users`
2. 角色管理页面：`/system/roles`
3. 只有管理员可以管理用户和角色
4. 用户只能编辑级别比自己低的用户

## 技术栈

- 后端：NestJS + TypeORM + PostgreSQL
- 前端：Vue 3 + TypeScript + Element Plus
- 认证：JWT
- 权限：基于角色的访问控制（RBAC）
